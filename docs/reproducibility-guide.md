# Reproducibility guide

Step-by-step from a fresh clone to byte-identical outputs. Read this if you want to *verify* — not just *trust* — the numbers in the manuscript.

## What "reproducible" means here

Three increasing levels of guarantee, each documented and independently checkable:

1. **Run-time deterministic.** Every script's outputs are deterministic given the inputs and the pinned environment. Re-running on the same machine produces identical bytes.
2. **Cross-machine reproducible.** With the pinned environment in [`requirements.txt`](../requirements.txt) (Python 3.11 + matplotlib 3.8.2 + numpy 1.26.4 + pandas 2.1.4 + scipy 1.11.4), outputs are byte-identical across Linux / macOS / Windows-WSL.
3. **Re-derivable from the public record.** The PubMed-derived corpus is regenerated by re-running the pre-registered query at NCBI E-utilities. Hit counts will drift over time as PubMed grows; the drift is reported in [`data/QUALITY_REPORT.md`](../data/QUALITY_REPORT.md) and is informational, not error.

Choose a level based on your goal. For peer review, level 2 is what you want; for ML training, level 3 is fine.

## Prerequisites

| Software | Pinned version | Why |
|---|---|---|
| Python | 3.11.x | Asyncio + typed-dict semantics used in scripts; tested on 3.11.4–3.11.9. |
| `make` (GNU) | 4.x | Convenience entry point. **Optional** — every target can be run as a direct `python` command (see below). |
| Git | ≥ 2.30 | For clone + verify. |

Operating-system notes:

- **Linux / macOS / WSL2:** install Python 3.11 via your package manager or `pyenv`; `make` is preinstalled or one `apt`/`brew` install away.
- **Windows (native):** Python 3.11 from python.org works; `make` is **not** native to Windows. Use the direct `python` invocations under "Without `make`" below, or install via `chocolatey install make` / `winget install GnuWin32.Make`. WSL2 also works perfectly.

## Quick start (3 commands)

```bash
git clone https://github.com/mikhaeelatefrizk/affect-labeling-review.git
cd affect-labeling-review
make all
```

`make all` does the following, in order:
1. `pip install -r requirements.txt` (idempotent)
2. Runs `meta-analysis/run_meta_analysis.py` → effect-size CSVs + forest/funnel figures
3. Runs `prisma/build_prisma.py` → PRISMA flow figure + counts text
4. Runs `supplementary/build_rob_figure.py` → risk-of-bias traffic-light figure
5. Runs `scripts/extract_included_list.py` → `data/screening/included_papers.csv`
6. Runs `scripts/build_derived_corpus.py` → `data/screening/derived_corpus.csv` (queries PubMed)
7. Runs `scripts/build_derived_screening_log.py` → `data/screening/derived_screening_log.csv`
8. Runs `scripts/build_quality_report.py` → `data/QUALITY_REPORT.md`
9. Runs `scripts/validate_screening_log.py` → schema + reconciliation check
10. Prints SHA-256 of every regenerated output

`make verify` additionally asserts no diff against the committed versions for the byte-deterministic outputs (excludes `derived_corpus.csv` and `derived_screening_log.csv`, which legitimately drift over time).

## Without `make`

```bash
git clone https://github.com/mikhaeelatefrizk/affect-labeling-review.git
cd affect-labeling-review
python -m venv .venv
.venv\Scripts\activate                  # on Windows
# source .venv/bin/activate              # on Linux/macOS
pip install -r requirements.txt
python meta-analysis/run_meta_analysis.py
python prisma/build_prisma.py
python supplementary/build_rob_figure.py
python scripts/extract_included_list.py
python scripts/build_derived_corpus.py
python scripts/build_derived_screening_log.py
python scripts/build_quality_report.py
python scripts/validate_screening_log.py
```

Each script is self-contained and prints `[OK] Wrote <path>` lines on success. Total runtime is ~3–5 minutes, dominated by the PubMed E-utilities calls (~30s for ESearch + ~2 min for EFetch on 3,900 records at the default unauthenticated rate limit).

## Faster PubMed runs

Get a free NCBI API key at https://www.ncbi.nlm.nih.gov/account/ and set:

```bash
export NCBI_API_KEY=your_key_here       # Linux/macOS
$env:NCBI_API_KEY="your_key_here"       # Windows PowerShell
```

The corpus build script auto-detects the env var and bumps the rate limit from 3 req/s to 10 req/s. Runtime drops to ~30 seconds total.

## Verifying byte-identical outputs

After a clean run:

```bash
make sha    # prints SHA-256 of every generated file
git diff --exit-code -- 'meta-analysis/*.csv' 'meta-analysis/*.txt' \
  'prisma/prisma_counts.txt' 'prisma/prisma_counts.csv' \
  'data/screening/included_papers.csv'
```

If `git diff` returns 0 (no output), your run reproduced the canonical bytes. If it returns 1 (diff present), inspect the diff and check:

1. Are you on Python 3.11.x? (`python --version`)
2. Are dependency versions pinned? (`pip freeze | sort`)
3. On Windows, did line endings differ? (`git config core.autocrlf` should be `true`).

Figures (`figures/*.png`, `*.pdf`) are excluded from the byte-identical check because matplotlib version differences produce visually identical but byte-different output. The PNGs and PDFs in the repo were generated with matplotlib 3.8.2; on a different version you'll get different bytes but the same plots.

## Containerised reproduction (optional)

A thin Dockerfile is not yet shipped (it would be redundant with the pinned `requirements.txt`), but if you need bit-perfect reproducibility across years, the recipe is:

```dockerfile
FROM python:3.11.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-deps -r requirements.txt
COPY . .
CMD make all
```

Pinning `--no-deps` and using a specific Python patch version (3.11.9) gives you a frozen environment that will reproduce identically in 5 years.

## Common pitfalls (see also [`troubleshooting.md`](troubleshooting.md))

1. **Wrong Python version.** Pyhton 3.10 misses some `typing_extensions` imports used in scripts. Use 3.11.
2. **Newer matplotlib.** matplotlib 3.10 produces visually identical but byte-different PNGs. The pinned 3.8.2 is what was used to commit the canonical figures.
3. **PubMed rate limit.** Without an API key, the unauthenticated rate limit is 3 req/s. Long pauses + the `WARN: batch starting at N failed; retrying` log message indicate rate-limiting. Set `NCBI_API_KEY`.
4. **Windows line endings.** Git's `core.autocrlf=true` on Windows normalises line endings on commit. If `make verify` flags `prisma_counts.txt` as different, ensure your local file is LF-terminated.
5. **Network unavailable.** `build_derived_corpus.py` requires internet to query PubMed. The other scripts are fully offline.

## What "make all" does NOT regenerate

- `manuscript/manuscript.md` — authored, not generated.
- `references.bib` — authored.
- `prereg/PROSPERO_preregistration.md` — authored.
- All files under `docs/` — authored.
- LICENSE files, CITATION.cff, .zenodo.json, README — authored.

If you change one of these, `make all` will *not* rebuild them. They are version-controlled directly.

## Debugging an unexpected diff

If `make verify` fails after a clean install on the pinned environment, the most common causes are:

1. **You modified an authored file.** Check `git status`.
2. **A script picked up a different version of a transitive dependency.** Run `pip freeze > /tmp/local-freeze.txt` and compare against the pinned versions in `requirements.txt`.
3. **A floating-point ULP-level difference.** Numpy uses platform-specific BLAS; on rare combinations (e.g., Apple Silicon Accelerate vs. x86 OpenBLAS) the last bit of a float64 in `extracted_effect_sizes.csv` may differ. The reported numbers (truncated to 3 decimal places in the manuscript) are unaffected.

If none of these apply, please open a [reproducibility issue](https://github.com/mikhaeelatefrizk/affect-labeling-review/issues/new?template=reproducibility.yml) with your platform, Python version, and the diff output.

## Outside the pinned environment

The pinned versions are conservative; on newer Python / numpy / pandas the scripts also run. Outputs may drift in non-substantive ways:

- **PNG bytes**: matplotlib version-sensitive.
- **CSV float precision**: numpy/scipy version-sensitive at the last ULP.
- **`derived_corpus.csv` row count**: PubMed-snapshot-sensitive (grows over time).

The numerical results in the manuscript (Hedges' *g* = −0.43, etc.) are robust to these drifts.
